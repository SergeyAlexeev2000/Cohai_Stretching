# Cohai Stretching — мастер‑план сайта

## 0. Подготовка и анализ

0.1. Собрать и зафиксировать все источники
- Скриншоты из приложения (регистрация, логин, правила, типы занятий).
- Скриншоты из Instagram:
  - Памятка новичку (4 шага).
  - Вакансии тренеров.
  - Отзывы.
  - Расписание групп (RU/RO).
  - Кейсы (беременность и др.).
  - Программа лояльности.
  - Прайс‑лист (группы, персональные, парки).
- Описание текущего backend‑проекта и БД.

0.2. Определить ключевые бизнес‑цели сайта
- Онлайн‑витрина студии + доверие (фото, кейсы, отзывы).
- Входная воронка в приложение / CRM (лиды с сайта).
- Объяснение продуктов: форматы тренировок, абонементы, цены.
- Навигация: как найти студию, карта, контакты.
- HR‑задачи: привлечение тренеров.

0.3. Решить, как сайт связан с существующим приложением
- Сценарий 1: сайт как «фронт» для нашего backend’a (запись на пробную, лиды).
- Сценарий 2: сайт только витрина + кнопки «Открыть приложение» / ссылки в сторы.
- Сценарий 3 (гибрид): сайт ведёт заявку, админ затем вручную заводит клиента в приложение.

## 1. Информационная архитектура сайта (структура страниц)

### 1.1. Главная страница
1.1.1. Герой‑блок
- Лого COHAI STRETCHING.
- Короткий слоган (из инсты / приложения).
- CTA‑кнопки:
  - «Записаться на пробную тренировку» (форма на сайте).
  - «Скачать приложение» (Google Play / App Store, когда будет).

1.1.2. «Для кого наш стретчинг»
- Блок с 3–4 тезисами (осанка, гибкость, здоровье спины, анти‑стресс).
- Кнопка «Памятка новичку» (ведёт на отдельную страницу/секцию).

1.1.3. «Форматы тренировок»
- Карточки: активный стретчинг, классический стретчинг, танцы (female styles), персональные, мини‑группы, пары, тренировки в парке, детские, мужские.
- Для каждого кратко: цель, кому подходит, пример результата.
- Ссылки на расписание и цены.

1.1.4. «Почему Cohai Stretching»
- Пункты: опыт тренеров, анатомия и биомеханика, безопасность, атмосфера студии.
- Можно частично взять формулировки из «Памятки новичку».

1.1.5. «Отзывы» (превью‑блок)
- Слайдер 3–5 отзывов из инсты с возможностью перейти на отдельную страницу с полным архивом.

1.1.6. «Программа лояльности» (тизер)
- Краткое описание механики «приведи друга — получи X тренировок».
- Кнопка на полное описание.

1.1.7. CTA‑блок внизу
- Повторный призыв «Записаться на пробную».
- Кнопки мессенджеров (WhatsApp/Telegram/Instagram‑DM).

### 1.2. Страница «Как начать / Памятка новичку»

1.2.1. Шаг 1. Определяем, с чего вы хотите начать
- Активный стретчинг / Классический / Танцы (female styles).
- Форма или кнопка «Написать администратору», чтобы подобрать формат.

1.2.2. Шаг 2. Выбираем формат занятий
- Индивидуальные / Группа / Мини‑группа / Пара.
- Линк на страницу с форматами/прайсом.

1.2.3. Шаг 3. Подбираем удобное время
- Описание процесса согласования времени (по телефону/чату).
- Ссылка на расписание.

1.2.4. Шаг 4. Приходите на пробное занятие
- Описание, что будет на первом занятии, какие ощущения.
- Упоминание, что пробное платное, но можно зачесть при покупке абонемента (если так в условиях).

1.2.5. FAQ новичка
- Какие вещи с собой (одежда, полотенце, вода, коврик).
- Противопоказания / рекомендации.
- Как отменить тренировки (8 часов / до 20:00 предыдущего дня).

## 2. Функциональные модули сайта

2.1. Модуль лид‑формы «Запись на пробное»
- Поля: имя, телефон, предпочитаемый формат, желаемое время, комментарий.
- Интеграция с backend: создание Lead в нашей БД.
- Уведомление администратору (email/Telegram — позже).

2.2. Модуль публичного расписания (интеграция с backend)
- API `GET /public/locations`, `GET /public/memberships`, `GET /public/schedule`.
- Фильтры по локации и типу занятия.
- Отображение в виде списка/таблицы.

2.3. Модуль отображения цен
- Статика в БД или конфиге, чтобы Анастасия могла править через админку в будущем.

2.4. Модуль отзывов
- Пока только вручную собранные отзывы.
- Позже — форма + модерация через админку.

2.5. Модуль карты и филиалов
- Таблица `locations` в БД.
- На фронте — маркеры с адресом, названием, ссылкой на расписание по локации.

2.6. Модуль «Вакансии»
- Форма отправки: имя, телефон, опыт, ссылка на соцсети, резюме (файл / ссылка).
- Пока можно просто отправлять на почту + писать запись в БД.

- Простая одностраничная версия (главная) + форма «Записаться на пробное».
- Разделы на странице:
  - Короткий блок «Как начать» (4 шага).
  - Базовое расписание (хотя бы текстом).
  - Основной прайс групповых тренировок.
  - Кратко о студии + контакты, карта.
- Интеграция формы лида с нашим backend’ом.
- Кнопки «Скачать приложение» / «Открыть в Google Play».

### 4.2. Этап 2 — Полноценный сайт v1

- Разнести всё по страницам (как в разделе 1).
- Подключить реальную структуру абонементов и расписания из БД.
- Оформить кейсы, отзывы, программу лояльности.
- Сделать RU/RO переключатель.

### 4.3. Этап 3 — Продвинутый функционал v2

- Личный кабинет клиента (в будущем — возможно связать с приложением).

- Небольшая админка для редактирования расписания, цен и контента.
- Автоматические уведомления администратору о заявках.
